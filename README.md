# Resource Booking API

## Кратко
Сервис бронирования переговорных комнат и оборудования с двумя вариантами реализации: FastAPI на Python и C#/ASP.NET Core. В проекте реализованы роли, аутентификация через JWT, контроль пересечений по времени, ограничения на бронирование и интерфейсы для локального запуска.

## Задача
Нужно организовать единый контур бронирования ресурсов, где система умеет:
- проверять доступность ресурса;
- запрещать пересекающиеся бронирования;
- учитывать роли пользователей;
- давать воспроизводимый локальный запуск в двух технологических вариантах.

## Структура репозитория
```text
fastapi-booking/         реализация на FastAPI
src/                     реализация на C#/ASP.NET Core
tests/BookingService.Tests  тесты для C#-части
.github/workflows/       автоматизация проверок
```

## Основные правила системы
- бронирования не должны пересекаться по времени;
- учитываются роли пользователей;
- часть действий требует авторизации через JWT;
- доступны интерфейсы для локального тестирования и документации API.

## Что улучшено
- в одном репозитории собраны две реализации одной предметной области;
- логика бизнес-ограничений отделена от интерфейса;
- проект удобен для сравнения архитектурных решений между Python и C#.

## Запуск
Запуск зависит от выбранной реализации:
- FastAPI-часть — через папку `fastapi-booking/`;
- C#-часть — через решение `BookingService.sln`.

## Ограничения
- для промышленного контура потребуются миграции, аудит действий и мониторинг;
- при росте числа пользователей понадобится отдельная стратегия масштабирования и блокировок;
- для реальной эксплуатации желательно вынести хранилище и конфигурацию в отдельные окружения.
