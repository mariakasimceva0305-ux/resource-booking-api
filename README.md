# Resource Booking API

Система бронирования переговорных и оборудования с двумя реализациями: `FastAPI` на Python и `ASP.NET Core` на C#. Поддерживаются роли, `JWT`, ограничения на бронирование, проверка пересечений по времени и `Swagger/OpenAPI`.

## Задача
Нужно предоставить API для бронирования ресурсов с учётом правил доступа, временных пересечений и ограничений на создание бронирований.

## Архитектура решения
```text
Пользователь
   │
   ▼
Аутентификация и роли
   │
   ▼
Проверка правил бронирования
   │
   ▼
Проверка пересечений по времени
   │
   ▼
Сохранение в базе
   │
   ▼
Ответ API
```

## Структура репозитория
- реализация на Python / `FastAPI`;
- реализация на C# / `ASP.NET Core`;
- схема хранения данных;
- документация `Swagger/OpenAPI`.

## Возможности
- аутентификация через `JWT`;
- роли и разграничение доступа;
- бронирование переговорных и оборудования;
- ограничения по времени и по типам ресурсов;
- защита от конфликтующих бронирований.

## Метрики
Для этого репозитория численные метрики качества не являются обязательными. Важнее показать:
- корректность проверки пересечений;
- сценарии доступа по ролям;
- покрытие основных пользовательских маршрутов.

## Запуск
Команды запуска зависят от выбранной реализации. В README нужно оставить отдельные блоки для Python и C#.

### FastAPI: локальная учётная запись администратора
Пароль администратора в коде не зашит. Для первого запуска в dev можно включить создание админа из переменных окружения (в каталоге `fastapi-booking`):

```bash
set BOOTSTRAP_ADMIN_ENABLED=true
set BOOTSTRAP_ADMIN_EMAIL=admin@example.com
set BOOTSTRAP_ADMIN_PASSWORD=your-secure-password
```

Без `BOOTSTRAP_ADMIN_PASSWORD` при `BOOTSTRAP_ADMIN_ENABLED=true` приложение не стартует. В тестах используется отдельный временный `DATABASE_URL` и те же переменные задаются в `tests/conftest.py`.
