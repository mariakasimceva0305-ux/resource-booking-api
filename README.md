# Resource Booking API

Система бронирования переговорных и оборудования с двумя реализациями: `FastAPI` на Python и `ASP.NET Core` на C#. Поддерживаются роли, `JWT`, ограничения на бронирование, проверка пересечений по времени и `Swagger/OpenAPI`.

## Назначение
API для бронирования ресурсов с учётом ролей, групп доступа, лимитов на число активных броней, проверки пересечений по времени и единообразного описания контракта через OpenAPI.

## Архитектура решения
```text
Пользователь
   │
   ▼
Аутентификация и роли
   │
   ▼
Проверка правил бронирования
   │
   ▼
Проверка пересечений по времени
   │
   ▼
Сохранение в базе
   │
   ▼
Ответ API
```

## Структура репозитория
- реализация на Python / `FastAPI`;
- реализация на C# / `ASP.NET Core`;
- схема хранения данных;
- документация `Swagger/OpenAPI`.

## Возможности
- аутентификация через `JWT`;
- роли и разграничение доступа;
- бронирование переговорных и оборудования;
- ограничения по времени и по типам ресурсов;
- защита от конфликтующих бронирований.

## Метрики
Для сервисного API важны корректность бизнес-правил (пересечения, роли, лимиты) и воспроизводимая проверка ключевых маршрутов; отдельные ML-метрики здесь не используются.

## Запуск

### Python (`FastAPI`)

Рабочий каталог: `fastapi-booking`.

```bash
cd fastapi-booking
python -m venv .venv
# Windows: .venv\Scripts\activate
# Linux/macOS: source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### C# (`ASP.NET Core`)

Из корня репозитория:

```bash
dotnet run --project src/BookingService.Api
```

Детали конфигурации и тестов — в структуре решения `src/` и `tests/`.

### FastAPI: локальная учётная запись администратора
Пароль администратора в коде не зашит. Для первого запуска в dev можно включить создание админа из переменных окружения (в каталоге `fastapi-booking`):

```bash
set BOOTSTRAP_ADMIN_ENABLED=true
set BOOTSTRAP_ADMIN_EMAIL=admin@example.com
set BOOTSTRAP_ADMIN_PASSWORD=your-secure-password
```

Без `BOOTSTRAP_ADMIN_PASSWORD` при `BOOTSTRAP_ADMIN_ENABLED=true` приложение не стартует. В тестах используется отдельный временный `DATABASE_URL` и те же переменные задаются в `tests/conftest.py`.

## Ограничения и развитие
- для промышленного использования обычно подключают миграции схемы БД, мониторинг и журналирование;
- для Python-части базовые сценарии регистрации, входа и бронирования покрыты тестами в `fastapi-booking/tests/`; при расширении домена имеет смысл добавить кейсы на сложные конфликты расписания и граничные права доступа.
